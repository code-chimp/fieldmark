## Challenges and Patterns

### Review churn (high frequency, expected by construction)
- Stories 1.4 / 1.6 / 1.11 / 1.13 / 1.14 went multi-round.
- **Project Lead framing:** review churn is a process smell but is not unexpected with non-idempotent agentic code generation, especially against a non-standard architecture (three parallel stacks against one canonical schema). Trial and error is expected.
- **Follow-up:** post-retro review-churn analysis to categorise recurrence patterns and surface candidate role/skill prompt fragments (Action Item **A1**).

### Build-tooling fragility
- `optimize-css.mjs` absorbed six review rounds: hardcoded pnpm store paths, `errorRecovery: true` silently swallowing LightningCSS errors, regex dedup breaking on CRLF, missing atomic write, no path-traversal guard, no validation of vendor fonts.
- Now hardened (atomic temp write, path guard, fatal-warning exit, `preinstall` pnpm guard, prebuild `check-sources` + `check-basecoat-classes`).
- Carried debt: duplicate token definitions in `_tokens.css` / `_ag-grid.css` vs Basecoat (Action Item **A6**).

### Same-shape bugs repeating across stacks
- Role selection picked non-canonical roles before canonical roles in both .NET *and* Django (Story 1.13 and again at 1.14 final rerun).
- `Role.Label()` Go default leaked raw strings; warning paths unreachable in .NET and Django due to symmetric over-filtering.
- Reviewer typically catches stack #2 after stack #1 is fixed.
- **Mechanical countermeasure:** canonical example in `fieldmark_shared/components/` first, then implement against the snapshot in each stack (Story 1.12's ActionButton pattern). Epic 2 amendments to Story 2.4 make this the rule for Phase-2 components.

### Tests that "skip gracefully" can silently bypass gates
- Story 1.14 hardened this four times: axe-core skipping when `npx` absent, sidebar PE test skipping when unauthenticated, font CLS guard executing in wrong scope.
- The skip wasn't wrong; the gate wasn't *enforced* anywhere. Resolved with mandatory CLS-capable lane (Chromium must throw) and a CSS-contract test that always runs.

## Significant Discovery — Architectural Principle Ratified

**Project Lead clarified during this retro:** each stack stands alone as a self-contained, idiomatic application. A native developer opening one stack must see every enum, DTO, DAO, handler, and test in its expected location.

- **Shared only via symlink:** the compiled design-system bundle (`fieldmark_shared/dist/fieldmark.css`) and vendored static assets (`htmx.min.js`, `ag-grid-community.min.js`, fonts). Full list.
- **Cross-stack invariants live as documentation contracts, not shared code:** a spec under `docs/reference/` or `docs/how-to/`, a native implementation per stack, and a per-stack conformance test.

**This principle changes the *form* of Epic 2 acceptance criteria, not the scope.** Stories 2.2, 2.4, 2.9, and 2.12 received AC amendments during this retro (see "Story AC Amendments Landed During Retro" below). The principle has been landed in the root [CLAUDE.md](../../CLAUDE.md) under a new section **Cross-Stack Architecture Principle**.

## Team Agreements (carrying into Epic 2)

1. **Code review is human-gated.** Reviewer finds, developer decides, human approves. Reviewer findings are surfaced, not silently auto-applied.
2. **Each stack stands alone.** Native enums, DTOs, DAOs, handlers per stack. Only the compiled design-system bundle and vendored static assets are symlinked.
3. **Cross-stack invariants live as documentation contracts.** A doc in `docs/reference/` (or `docs/how-to/` for patterns) is the source of truth; native implementation per stack; per-stack conformance test.
4. **Canonical example first, then implement per stack.** Story 1.12's ActionButton pattern is the template.
