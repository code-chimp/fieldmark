# Project Classification

| Attribute           | Value                                                                                                                                                                                                                                                                                              |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Project type        | Web application (server-rendered MPA with HTMX interactivity and AG Grid islands)                                                                                                                                                                                                                  |
| Domain              | Enterprise workflow simulation — construction compliance & inspection management                                                                                                                                                                                                                   |
| Domain note         | Realistic but simulated; no real regulatory exposure (FDA, NERC, PE-stamp). The audience for the artifact is the actual primary user; in-app personas exist to make workflows credible.                                                                                                            |
| Complexity          | Medium-high. Driven by three-stack symmetry, four aggregate roots with state machines, server-evaluated rules engine and compliance scoring, and ADR-locked architectural guardrails (no CQRS, no repositories, no client state, infrastructure-owned domain schema) — _not_ by regulatory burden. |
| Project context     | Greenfield. Planning artifacts (project brief, domain model, ADRs, UX guide) are locked; product code is in early implementation across all three stacks.                                                                                                                                          |
| Primary audience    | Mid-to-senior engineers, tech leads, and architects evaluating HTMX vs. SPA defaults. The talk is the deliverable; the application is the demonstration.                                                                                                                                           |
| Success orientation | Architectural thesis validation, with SPA-equivalent interaction smoothness as a falsifiable UX target.                                                                                                                                                                                            |
